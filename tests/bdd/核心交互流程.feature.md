# 漂亮饭 · 核心交互流程 BDD（Flutter + RN 共用）

> 套 `bdd-loop-harness-verify-cycle` pattern：`.feature.md`(Gherkin 人读) × `maestro/flow-*.yaml`(机器跑) × `run_bdd.sh`(≤100 行 runner)。
> 一套 flow 跑两端 → A/B benchmark。图片 = `test-assets/ganguoxia.jpg`（干锅虾，已 addmedia 进模拟器相册）。

## Feature: 拍一张饭 → AI 算值不值得发 → 出炫的图文编排

### 背景
- **Given** 模拟器为 iPhone 17 Pro（booted）
- **And** 相册已注入干锅虾图（`ganguoxia.jpg` 为相册第一张）
- **And** app 已安装

### Scenario: 完整流程 · 选图 → 预设 → AI 算 → 图文编排
- **Given** app 启动，看到占位框「选一张你的漂亮饭」
- **When** 我点占位框
- **And** 系统相册弹出，我选第一张（干锅虾）
- **Then** 回到主界面，饭图已显示 + 出现预设条 + 「AI 算这顿值不值得」按钮
- **When** 我选「胶片框」预设
- **And** 我点「AI 算这顿值不值得」
- **Then** 出现 loading（真调 M3 vision，非 mock）
- **And** 数秒内出现热量标签：菜品名 + kcal 大数字 + 蛋白/碳水/脂肪 + 「不内疚」tags
- **And** 图文编排「有炫感」（标签浮在美化后的图上，胶片框 + 日期戳）

### Scenario: 检测质量验收（M3 真识别）
- **When** 对干锅虾图执行识别
- **Then** 菜品名含「虾」（干锅虾 / 干锅虾配时蔬）
- **And** kcal 在 200~600 区间（合理正餐量）
- **And** tags 含「高蛋白」（虾是高蛋白）
- **And** 输出严格 JSON（前端可解析不崩）

### Scenario: 失败降级
- **When** M3 返回错误 / 网络断
- **Then** 显示 ⚠️ 错误文案，不崩溃，不卡 loading
